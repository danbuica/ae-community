# Generate Brand Style Guide

Read a client's onboarding submission from `data/clients/{slug}/onboarding.json` and generate a comprehensive markdown brand style guide. Save it as `data/clients/{slug}/style-guide.md`.

## Instructions

1. **Find the client folder**
   - If the user provided a client name as an argument (`$ARGUMENTS`), look for a matching folder in `data/clients/` (e.g., "ardona" → `data/clients/ardona/`)
   - If no argument given, list all folders in `data/clients/` and pick the most recently modified `onboarding.json`
   - Read `onboarding.json` and parse all fields.

2. **Generate the style guide**
   Write a polished, professional markdown document with the following sections. Use the submission data to populate every section — never leave a section empty. Where data is missing, write sensible defaults based on the brand's industry and archetypes.

   ```
   # [Business Name] Brand Style Guide

   > Generated by Brand Bounce · [current date]

   ---

   ## 1. Brand Identity

   ### Brand Overview
   - Business name, industry, elevator pitch (full text)
   - Unique selling proposition
   - Key products/services (bulleted list)

   ### Brand Archetypes
   For each selected archetype: name, short description of what it means for this brand's communication style.

   ### Brand Voice
   Full brand voice description paragraph, followed by:
   - **Do:** 3 bulleted concrete examples of on-brand writing
   - **Don't:** 2 bulleted examples of off-brand writing

   ---

   ## 2. Visual Identity

   ### Color Palette
   For each brand color: a markdown table row with color name, hex code, role (primary/secondary/accent), and suggested use cases.
   Include a "Copy-paste CSS variables" code block:
   ```css
   :root {
     --color-primary: #hex;
     --color-secondary: #hex;
     /* ... */
   }
   ```

   ### Logo Usage
   - Logo URL (if uploaded): display with `![logo](url)`
   - Minimum clear space guidance
   - Dos and don'ts (don't stretch, don't place on clashing backgrounds, etc.)

   ### Typography Guidance
   Based on the brand's tone keywords and archetypes, recommend:
   - Primary font pairing (Google Fonts, free)
   - Heading vs body usage
   - Example CSS `font-family` snippet

   ---

   ## 3. Target Audience

   ### Audience Persona
   Full audience description paragraph.
   - Age range, locations, interests (bulleted)
   - Key pain points
   - What they respond to in content

   ---

   ## 4. Content Strategy

   ### Content Pillars
   For each pillar: name in bold, one-sentence description below.

   ### Platform Strategy
   For each selected platform: name, recommended posting frequency (posts/week), top posting times (if available), and 2-3 content format recommendations.

   ### Hashtag Strategy
   For each hashtag group: pillar name as subheading, then the hashtag list formatted as inline code (`#tag`).

   ### Content Formats (Priority Order)
   Numbered list of formats in the order the client specified.

   ### Topics to Avoid
   Bulleted list.

   ---

   ## 5. Goals & KPIs

   ### Primary KPIs
   Bulleted list of selected KPIs with a one-line description of how to measure each.

   ### Budget & Timeline
   - Monthly ad budget tier
   - Approval workflow
   - Preferred communication channel
   - Key upcoming dates (table: Date | Event)

   ---

   ## 6. Social Media Handles

   Table: Platform | Handle — for all handles the client provided.

   ---

   ## 7. Brand Checklist

   A quick-reference checklist for the team:
   - [ ] Logo approved in all formats (PNG, SVG, dark/light variants)
   - [ ] Color palette added to design tools (Figma / Adobe)
   - [ ] Font licenses confirmed
   - [ ] Brand voice guide shared with copywriters
   - [ ] Hashtag groups saved in scheduling tool
   - [ ] Platform profiles updated with bio, link, and profile image
   - [ ] First content calendar drafted using the pillars above
   - [ ] KPI tracking dashboard set up

   ---

   *This style guide was auto-generated from the BrandBounce onboarding brief. Last updated: [date]*
   ```

3. **Save the file**
   Write the completed markdown to `data/clients/{slug}/style-guide.md`.

4. **Report back**
   Tell the user:
   - Which client was used (business name + slug)
   - The path of the saved file
   - A short summary of what the brand looks like (2-3 sentences)
