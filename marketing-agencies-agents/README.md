# Marketing Agencies Agents

Agents for running a marketing agency's day-to-day output: onboarding a client, turning their brief into a brand style guide, planning content, generating images, generating ads, and repurposing blogs into social posts.

Each agent is a **Claude Code slash command** — a markdown file in `.claude/commands/` that Claude Code exposes as `/command-name` when you open this folder. Some agents also call small Python helper scripts in `scripts/` to hit the Gemini image API.

## Agents

### 1. Client onboarding + `/generate-style-guide`
This one has **two parts**:

- [`brandbounce-onboarding/`](./brandbounce-onboarding) — a Next.js app where a client fills in a 7-step form (brand identity, voice, visuals, audience, platforms, KPIs, handles). Submissions are saved as JSON under `data/clients/{slug}/onboarding.json`.
- `/generate-style-guide` — reads that JSON and writes a polished brand style guide to `data/clients/{slug}/style-guide.md`.

Together they take a client from zero → a reusable brand style guide that every other agent below consumes.

### 2. `/content-planner`
Generates a full week of platform-specific content (Mon–Sun) from the client's style guide. Writes ready-to-post captions, hashtags, and visual descriptions. Saves to `data/clients/{slug}/content-plans/week-{date}.md`.

### 3. `/generate-images`
Reads a content plan and generates branded Instagram images (1080×1080) via Gemini. Calls `scripts/generate_instagram_images.py`. Outputs to `data/clients/{slug}/images/{plan-date}/`.

### 4. `/generate-ads`
Creates 5 ad copy variations (pain point, social proof, curiosity, direct offer, story hook) with feed + story images for each. Calls `scripts/generate_ad_creatives.py`. Outputs to `data/clients/{slug}/ad-creatives/{date}-{offer-slug}/`.

### 5. `/repurpose-content`
Takes a blog post (file, URL, or inline text) and turns it into Instagram post, Instagram story, LinkedIn, and Facebook versions with branded images. Calls `scripts/repurpose_content.py`. Outputs to `data/clients/{slug}/repurposed/{date}-{title-slug}/`.

## Setup

### Prerequisites
- [Claude Code](https://claude.com/claude-code) installed (`npm install -g @anthropic-ai/claude-code`)
- **Python 3.10+** (only if you want to run the image-generating agents 3, 4, 5)
- **Node.js 20+** (only for the onboarding app in agent 1)
- A **Gemini API key** — get one at [aistudio.google.com](https://aistudio.google.com/app/apikey)

### 1. Get this folder onto your machine
Either clone the full `ae-community` repo or download just `marketing-agencies-agents/` (see the root README for options). Open a terminal inside this folder.

### 2. Set your Gemini API key
This folder ships with a `.env.example`. Copy it to `.env` and fill in your key:
```bash
cp .env.example .env
# then edit .env and set GEMINI_API_KEY=your_actual_key
```
Before running the image-generating agents, load the file into your shell:
```bash
set -a && source .env && set +a
```
`.env` is gitignored so your key never lands on GitHub.

### 3. (Agent 1 only) Install and run the onboarding app
```bash
cd brandbounce-onboarding
cp .env.local.example .env.local   # then put GEMINI_API_KEY inside
npm install
npm run dev
```
The form runs at `http://localhost:3000`. When a client submits, the JSON is written to `brandbounce-onboarding/src/data/submissions/`. Copy or symlink the submission into `data/clients/{slug}/onboarding.json` before running `/generate-style-guide`.

### 4. Open Claude Code in this folder
```bash
cd marketing-agencies-agents
claude
```
Claude Code picks up `.claude/commands/*.md` automatically — all five slash commands will be available. Claude will install any Python dependencies the scripts need on first run.

## Running the agents

Inside Claude Code, from this folder:

```
/generate-style-guide <client-slug>
/content-planner <client-slug>
/generate-images <client-slug>
/generate-ads <client-slug> --offer "description of the offer"
/repurpose-content <client-slug> --url https://example.com/blog-post
```

Typical flow for a new client:
1. Client fills out the onboarding form → `onboarding.json` lands under `data/clients/{slug}/`
2. `/generate-style-guide {slug}` → produces `style-guide.md`
3. `/content-planner {slug}` → weekly plan
4. `/generate-images {slug}` → images for that week's Instagram posts
5. `/generate-ads {slug} --offer "..."` and `/repurpose-content {slug} --file blog.md` as needed

## Data layout

```
marketing-agencies-agents/
├── .claude/commands/                   # the five slash commands
├── scripts/                            # Python helpers called by the commands
├── brandbounce-onboarding/             # Next.js onboarding form (agent 1, part A)
└── data/
    └── clients/{slug}/
        ├── onboarding.json             # raw form submission
        ├── style-guide.md              # generated by /generate-style-guide
        ├── content-plans/              # /content-planner output
        ├── images/                     # /generate-images output
        ├── ad-creatives/               # /generate-ads output
        └── repurposed/                 # /repurpose-content output
```

## Adding a new agent

Drop a new markdown file into `.claude/commands/` (this becomes the slash command). If it needs to shell out for something heavy (image generation, API calls), add a Python helper in `scripts/` and have the command invoke it. Then add an entry to the list above.
